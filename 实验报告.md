融合策略:**NMS融合**

```python
# 对每个类别进行NMS融合
for cat_id, cand in per_cat_candidates.items():
    if len(cand["boxes_xyxy"]) == 0:
        continue
    # 合并该类别的所有框
    boxes_xyxy = np.vstack(cand["boxes_xyxy"])
    scores = np.hstack(cand["scores"])
    # NMS
    keep = nms(
        torch.from_numpy(boxes_xyxy).float(),
        torch.from_numpy(scores).float(),
        iou_threshold
    ).numpy()
```

# 失败案例分析

使用COCO数据集训练一轮后，推理一张图片的结果:

##### (1)box_threshold=0.25 text_threshold=0.25:![](C:\Users\ASUS\AppData\Roaming\Typora\typora-user-images\image-20260227191455961.png)

##### (2)box_threshold=0.85 text_threshold=0.85:![image-20260227191708222](C:\Users\ASUS\AppData\Roaming\Typora\typora-user-images\image-20260227191708222.png)

### 1. 训练配置中的潜在问题

- **训练轮次太少**：为了训练的速度设置epochs = 1，只训练了1个epoch。对于检测模型微调，尤其需要适应新数据分布，1个epoch往往不足以让模型充分收敛，可能导致特征提取和文本-视觉对齐仍停留在预训练状态。
- **模型结构简化**：对比原始GroundingDINO，为了训练的速度，将配置中的enc_layers 和 dec_layers 从6减少到4，可能会降低模型容量，影响对复杂场景（如同时检测狗和猫）的建模能力。
- **数据增强调整**：data_aug_scales 从较大的范围（480~800）缩小到320~544，且 data_aug_max_size 从1333降至800，这有助于降低计算量，但可能影响模型对多尺度目标的鲁棒性。
- **学习率与优化**：lr=0.0002 是基础学习率，lr_backbone=0.0001 对骨干网络稍低。1个epoch可能无法使模型有效更新。

### 2. 推理环节的可能原因

- **文本提示格式**：使用的 TEXT_PROMPT = "a dog.a cat." 包含句点。GroundingDINO通常期望用点或逗号分隔多个类别，但内部处理可能将句点视为普通字符，导致模型将整个字符串“a dog.a cat.”当作一个整体描述，而非两个独立类别。这可能导致模型输出一个包含两个名词的短语，并错误地附带 [SEP]（这是BERT分词时的特殊标记，但不应出现在最终输出中）。
- **模型权重问题**：推理时加载的权重是 checkpoint_best_regular.pth，但训练只进行了1个epoch，该权重可能并未达到最佳状态，甚至可能是未完全保存的中间结果，导致推理行为异常。
- **后处理逻辑**：predict 函数返回的 phrases 可能直接从模型输出中提取，如果模型未能正确分离类别，就会将 [SEP] 也当作短语的一部分输出。

### 3. 出现 [SEP] 和高置信度1.00的可能原因

- [SEP] 是BERT内部用于分隔句子的特殊标记，正常情况下模型解码时不会输出它。但当模型将整个输入视为一个长文本描述时，可能错误地将 [SEP] 保留在输出中，表示文本结束。
- 置信度1.00极不正常，说明模型可能对图像整体做出了一个“默认”预测（例如背景或全图），而非检测具体目标。这通常发生在训练不足或文本-视觉对齐失效的情况下。

### 4. 潜在解决方案

- **延长训练周期**：至少训练12个epoch以上，确保模型充分收敛。
- **检查文本提示格式**：尝试将句点改为空格或逗号，例如 "a dog . a cat ." 或 "dog . cat ."，并确保与训练时的类别格式一致。
- **验证训练数据标注**：确保训练数据中“dog”和“cat”的标注框正确，且类别名称与推理时使用的文本一致。
- **简化测试**：先用预训练权重（不微调）直接推理同一张图片，看是否能正确检测狗和猫，以排除模型结构或代码本身的问题。
- **监控训练损失**：观察训练过程中损失是否下降，以及验证集上的mAP是否提升，确保模型真正学到了新知识。
